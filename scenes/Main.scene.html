本教学实例的目录结构可能是这样的！

这里我们需要关注两个文件夹，config 和 resource ！

config 文件夹是设置文件存放的位置，随着游戏的进行，一些存档文件也会存在这里。

然后我们进入 resource 目录！

继续进入 app 目录！

我们就可以在此文件下，进行开发了。

avgengine 文件夹是引擎的主逻辑源代码，包括控制代码以及样式文件，以及一些依赖等等。

bgm 文件夹是你们懂得，背景音乐，支持的格式可多了，只要在浏览器能播放的声音，这里都是可以的 MP3 WAV OGG等。

此时我们又看见了 一个 config 文件夹，这个在本教程的情况下是用不到的（这里的 config 在另外一种开发模式下会用到。），可以暂时忽略。

images 文件夹里面存放着图片，主要是主界面和按钮之类的所需要的图片。

scense 文件夹里面存放着游戏相关的场景，图片等等，我们游戏按照一个怎样的情节来展开都在这里的代码来体现。

sound 文件夹里面存放着音效文件，一些简短的音效需要存放在这里。

style 文件夹里面存放游戏的自定义样式。

我们的引擎其实就是一个个网页，所用的技术也并没有多么难，就是一些基础的前端技术。

我们进入的游戏第一个界面就是写在 index.html 这个文件里面的。

除了介绍的这些文件夹，其他的文件和文件夹大家可以暂时忽略。

但是忽略不代表他们没有用，如果你的对网页前端技术有所涉猎，你应该能明白这些都是做什么的。
